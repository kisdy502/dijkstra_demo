# Dijkstra 与 A* 路径规划

## Dijkstra 算法核心步骤

1. **初始化**
   - 所有节点的距离设为无穷大，前驱节点设为无效值（如 {-1, -1}），起点的距离设为 0
   - 将起点 (距离=0, 坐标) 放入优先队列（小顶堆）

2. **主循环：从优先队列取出最小距离节点**
   - 弹出堆顶元素，获取当前节点 `(r, c)` 及其距离 `distance`
   - **惰性删除**：若 `distance > dist[r][c]`，说明是旧记录，跳过本轮
   - 计数器 `round++`，记录本轮取出的节点

3. **检查是否到达目标**
   - 若当前节点 `(r, c)` 等于目标节点 `(goal.row, goal.col)`，结束搜索

4. **遍历当前节点的所有邻居（4 方向：上下左右）**
   - 计算邻居坐标 `(nr, nc) = (r + dr, c + dc)`
   - 检查邻居是否可通行（非越界、非障碍物），不可通行则跳过
   - **松弛比较**：`newDist = dist[r][c] + map.grid[nr][nc]`
     - 若 `newDist < dist[nr][nc]`：更新 `dist[nr][nc] = newDist`，记录前驱 `prev[nr][nc] = {r, c}`，将 `(newDist, nr, nc)` 加入优先队列
     - 若 `newDist >= dist[nr][nc]`：跳过，不更新

5. **不可达判断**
   - 优先队列为空或目标节点的距离仍为无穷大，说明起点和目标不连通，路径不存在

6. **回溯路径**
   - 从目标节点开始，通过前驱数组 `prev` 反向追溯到起点，得到最短路径

### 关键机制：惰性删除

同一个节点可能被多次加入优先队列（每次找到更短路径时）。弹出时通过比较 `distance > dist[r][c]` 跳过旧记录，避免了从堆中删除元素的昂贵操作。

---

## A* 算法核心步骤

与 Dijkstra 基本一致，唯一区别在于**优先队列的排序依据**：

- Dijkstra：按 `dist`（已走过的真实距离）排序
- A*：按 `dist + 启发值`（已走过距离 + 到目标的估计距离）排序

### 启发函数要求

启发函数必须满足 **可接受性**：对任意节点的估计代价不超过实际最小代价。在四连通网格中使用曼哈顿距离满足此条件，保证 A* 找到最短路径。

### A* vs Dijkstra 的关系

- A* 不改变搜索的**完备性**（能否找到路径）和**最优性**（路径是否最短）
- A* 仅改变**探索顺序**，让搜索优先向目标方向推进，减少无效探索
- 在障碍物密集的环境（如 SLAM 地图）中，A* 的优势可能被启发函数计算开销抵消
